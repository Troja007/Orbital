# 2026-07-02 Continuous Improvement Model

## Decision

Treat this repository as a GitHub-synced knowledge and memory system, not only as a file store for SQL and scripts.

Every useful query, script, catalog comparison, endpoint investigation, context import, and analyst correction should improve future quality and speed when it produces reusable knowledge.

## Scope

Reusable knowledge may be stored as:

- project context
- product context
- query-method memory
- sanitized catalog result profiles
- notes and memory summaries
- installable Codex skills
- helper scripts and templates

## Reason

The project goal is that future Codex workspaces and users start from prior outcomes. Work should become faster and more accurate as the repository accumulates validated methods, terminology, source mappings, catalog context, and interpretation guidance.

## Boundaries

- Store reusable method knowledge, not raw endpoint evidence.
- Store sanitized lessons, not raw chat transcripts.
- Require user validation before saving learned query-method memory from an investigation.
- Keep credentials, hostnames, IP addresses, GUIDs, tenant data, raw API responses, and endpoint result rows out of GitHub.
