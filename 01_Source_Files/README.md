# Source Files

Store original or exported source material here. Prefer read-only handling of these files during analysis.

Use subfolders for Orbital query sources, script sources, catalog templates, and API references.

Within the Orbital repo source folder, `Catalog_queries` and `Catalog_scripts` are 1:1 copies from the Orbital product catalog. They are source references only and should not be edited in place.

The Orbital repo source folder also contains `custom_queries` and `custom_scripts`. These are the user's personal work folders. Treat them as source material unless the user explicitly asks to edit or update the GitHub-tracked files.

When copying source files into this folder, record the original location, content date, and copy date in the relevant notes.
