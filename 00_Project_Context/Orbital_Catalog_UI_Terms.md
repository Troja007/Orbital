## Meaning Of ID

In the Orbital Catalog UI, `ID` is the catalog item identifier used by Orbital to reference a query or script.

`ID` is different from the visible display name:

- `Name` is the human-readable catalog title.
- `ID` is the product/catalog reference key used in routes and actions.

The `ID` appears in the Catalog table, in the detail drawer, and in the `Use query` / `Use script` route.

## Project Rules

When referencing Orbital catalog entries:

- Use `ID` as the stable product/catalog reference key.
- Use `Name` as the human-readable title only.
- Preserve `ID` values exactly, including prefixes such as `org:`.
- Do not infer ownership from `Name`; use `Updated by`, `Created by`, folder location, and ID pattern together.
- Treat stock Cisco catalog entries and organization/custom entries as separate source classes.
- When exporting or documenting catalog entries, record both `Name` and `ID`.
- Before pushing context to a public repository, redact organization-specific catalog names, user attribution, opaque `org:` IDs, and routes.
