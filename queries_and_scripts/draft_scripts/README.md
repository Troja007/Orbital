# Script Drafts

Working area for script drafts, test variants, validation scripts, adapted catalog scripts, and investigation-specific response-action scripts that should live near the query/script source tree but are not yet promoted to personal reusable custom script work.

Use clear file names that describe platform, purpose, and status where useful, for example:

- `draft_windows_<purpose>.py`
- `test_cross_platform_<purpose>.py`
- `validation_linux_<purpose>.py`

Do not store raw endpoint results, hostnames, usernames, IP addresses, GUIDs, tenant identifiers, Job IDs, or credentials in script files.

Promote a script into `../custom_scripts/` only when the user explicitly wants it stored as personal reusable custom script work.
